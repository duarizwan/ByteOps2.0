---

## Summary

Replace the existing split-layout graph debugger (`RunsPage` + `GraphCanvas` always visible) with a plain-English Action Center feed. Users see pending approvals pinned at top (amber), failed actions next (red), and a date-grouped history of completed actions below. The graph moves to a slide-in `TraceDrawer` opened via "trace →" on any card. Two backend endpoints (`approve`, `cancel`) are already implemented — this plan is **frontend-only**.

### `summarizeAction` (`lib/summarize-action.ts`)

Pure function — no side effects, fully testable.

```typescript
function summarizeAction(run: AgentRun): ActionSummary

Steps:
1. Find first step where step_type === "tool_call"
   AND WRITE_VERBS.has(step.name.split("_")[0]).
2. Look up (run.intent, step.name) in the summary table (data-model.md).
3. Extract named fields from step.input with graceful fallbacks.
4. Return { summary, detail }.
5. If no matching step: return { summary: "${run.intent} action completed", detail: humanize(step.name ?? run.intent) }
```

### `AwaitingApprovalCard` — Approve / Cancel

```typescript
// Approve
const handleApprove = async () => {
  setApproving(true);
  try {
    await authFetch(`/api/agent-runs/${run.id}/approve`, { method: "POST" });
    onApproved(run.id);  // parent removes card from pending list
  } catch (e) {
    // 409 = already actioned; treat as stale, also remove
    if (e.status === 409) onApproved(run.id);
    else setError("Could not approve — please try again.");
  } finally {
    setApproving(false);
  }
};
```

Same pattern for Cancel, using `/cancel` endpoint and `onCancelled` callback.

### `NeedsFixingCard` — Error Classification

```typescript
function classifyError(error: string | null): ActionErrorCategory {
  if (!error) return "unknown";
  if (/401|403|token|expired|unauthorized/i.test(error)) return "auth";
  if (/timeout|503|api error/i.test(error)) return "timeout";
  return "unknown";
}
```

CTA rendering:
- `auth` → `<a href="/settings">Reconnect {platform} →</a>`
- `timeout` → `<button onClick={handleRetry}>Retry →</button>` (disabled if retry not implemented yet)
- `unknown` → no CTA

### `FilterTabs` — View Filtering

```typescript
function categorize(runs: AgentRun[], tab: FilterTab): CategorizedRuns {
  const visible = runs.filter(r => !dismissedIds.has(r.id));
  if (tab === "pending") return { pending: visible.filter(isPending), failed: [], history: [] };
  if (tab === "failed")  return { pending: [], failed: visible.filter(isFailed), history: visible.filter(isFailed) };
  return {
    pending: visible.filter(isPending),
    failed:  visible.filter(isFailed),
    history: visible.filter(isHistory),
  };
}
```

## Risks & Follow-ups

1. **`step.input` field names may differ from spec** — `summarizeAction` must have broad null-safe fallbacks. TDD against real run data is the mitigation; the lookup table in data-model.md reflects the spec intent, not verified production payloads.
2. **Stale `waiting_approval` cards** — the 30 s polling interval means a card approved elsewhere (e.g. another browser tab) can linger. The 409 handling on Approve/Cancel mitigates the bad state; a shorter poll or WebSocket would be the clean fix (future).
3. **`runs-page.tsx` left in place** — two implementations of the same route coexist briefly. The old file should be deleted in the first follow-up PR after QA passes.
