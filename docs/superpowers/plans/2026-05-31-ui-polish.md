# UI Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add chat progress stage labels, ActionCard status badges, graceful error states in chat, and skeleton loading for Action Center and trace view — all frontend-only changes that make ByteOps feel production-ready.

**Architecture:** Four independent UI layers each modifying one or two files. Stage labels hook into the existing SSE event stream by adding a `stage` field to the `Message` interface. Error states reuse the existing `classifyError` utility (expanded). Status badges and trace button styling are additive changes to `ActionCard`. Skeletons use the existing `animate-pulse` pattern from `context-panel.tsx`.

**Tech Stack:** React 19, Next.js 16 App Router, Tailwind CSS v4, Lucide icons, custom inline styles (no shadcn/ui). All components are in `frontend/src/`. Tests verified by running the dev server manually.

  Open `frontend/src/lib/classify-error.ts`. Replace the entire file:
  ```typescript
  import type { ActionErrorCategory } from "./action-center-types";

  const AUTH_RE        = /401|403|token|expired|unauthorized/i;
  const TIMEOUT_RE     = /timeout|503|api error/i;
  const OAUTH_RE       = /oauth|not connected|not authenticated|integration.*required|reconnect/i;
  const NO_RESULTS_RE  = /no results|no emails|nothing found|empty result/i;
  const MCP_RE         = /mcp|tool.*unavailable|service.*unavailable|tool.*error/i;

  export function classifyError(error: string | null): ActionErrorCategory {
      if (!error) return "unknown";
      if (OAUTH_RE.test(error))       return "oauth_missing";
      if (NO_RESULTS_RE.test(error))  return "no_results";
      if (AUTH_RE.test(error))        return "auth";
      if (TIMEOUT_RE.test(error))     return "timeout";
      if (MCP_RE.test(error))         return "mcp_unavailable";
      return "unknown";
  }
  ```
  Note: `oauth_missing` check comes before `auth` because OAuth errors may also contain auth keywords.
